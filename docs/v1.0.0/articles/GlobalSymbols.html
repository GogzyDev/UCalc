Global Symbols

UCalc allows you to expose additional global symbols that can be accessed from any expression.

This is done by creating a static class and marking it with the UCalcGlobalContext attribute.

All supported static members of that class will automatically become available inside expressions.

Basic Example

[UCalcGlobalContext]
public static class DamageConstants
{
    public const float CriticalMultiplier = 2f;
    public const float HeadshotMultiplier = 1.5f;
}

This exposes the symbols under the class name.

Example usage in expressions:

DamageConstants.CriticalMultiplier

Custom Path

You can specify a custom path in the attribute:

[UCalcGlobalContext("Combat.DamageConstants")]
public static class DamageConstants
{
    public const float CriticalMultiplier = 2f;
}

This makes the symbols available under the specified namespace-like path:

Combat.DamageConstants.CriticalMultiplier

This is useful for organizing symbols into logical groups.

No Prefix

If the path is set to an empty string, symbols are exposed without a prefix.

[UCalcGlobalContext("")]
public static class DamageConstants
{
    public const float CriticalMultiplier = 2f;
}

Now the symbols can be used directly:

CriticalMultiplier

Supported Members

The following static members are exposed as symbols:

  • fields

  • properties

  • methods

These members can then be used inside expressions just like built-in symbols.

Example:

Mathf.Clamp(value, 0, DamageConstants.CriticalMultiplier)

When Are Global Symbols Discovered?

Global symbol providers are automatically discovered when the system initializes.

Any static class marked with UCalcGlobalContext will be registered and made available to expressions.
