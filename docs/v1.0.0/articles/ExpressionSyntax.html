Expression Syntax

UCalc expressions are compiled into .NET expression trees and support a subset of familiar C# syntax designed to be intuitive and efficient.

This page lists all currently supported language features.

Literals

Expressions can contain literal values.

Numeric literals

10  
3.14  
-5

Numeric literals are parsed using the configured culture.

Boolean literals

true  
false

Null literal

null

Default literal

The default keyword resolves to the most appropriate type depending on context.

Example:

value = default

If value is float, the expression resolves default to:

0f

Identifiers

Identifiers refer to symbols available in the expression context.

These may include:

  • expression arguments
  • owner members
  • dependencies
  • global symbols

Example:

health  
damageMultiplier  
enemy.Health  
Mathf.PI

Autocomplete suggestions are provided in the editor while typing.

Unary Operators

Unary operators operate on a single value.

Supported unary operators:

Operator Description
+ Unary plus
- Numeric negation
! Logical NOT
~ Bitwise NOT

Example:

-health  
!isAlive  
~mask

Binary Operators

Binary operators operate on two values.

Arithmetic

Operator Description
+ Addition
- Subtraction
* Multiplication
** Exponentiation
/ Division
% Modulo

Example:

health * damageMultiplier + bonus

Comparison

Operator Description
== Equal
!= Not equal
> Greater than
< Less than
>= Greater or equal
<= Less or equal

Example:

health <= 0

Logical operators

Operator Description
&& Logical AND
|| Logical OR

Example:

isAlive && stamina > 0

Bitwise operators

Operator Description
& Bitwise AND
| Bitwise OR
^ Bitwise XOR
<< Bit shift left
>> Bit shift right

Example:

mask & layer

Function Calls

Functions can be invoked using standard call syntax.

functionName(arg1, arg2)

Example:

Mathf.Clamp(value, 0, 1)

Functions support overloads, and the best overload is selected based on argument types.

Autocomplete suggestions show available overloads in the editor.

Constructors

Objects can be created using the new keyword.

new TypeName(arg1, arg2)

Example:

new Vector3(x, y, z)

Constructors support overload resolution, similar to function calls.

Indexing

Index expressions allow accessing elements using indexers.

collection[index]

Example:

inventory[0]  
grid[x]

Indexers are resolved based on available indexer properties.

Ternary Operator

Conditional expressions use the ternary operator:

condition ? valueIfTrue : valueIfFalse

Example:

health > 0 ? aliveScore : deadScore

Optional else branch

The else branch is optional.

This allows ternary expressions to be used for conditional actions.

Example:

health <= 0 ? Die()

Operator Precedence

Operators follow standard precedence rules similar to C#:

  1. Unary operators
  2. Multiplication / division
  3. Addition / subtraction
  4. Comparisons
  5. Logical operators
  6. Ternary operator

Parentheses can be used to explicitly control evaluation order.

Example:

(a + b) * c
