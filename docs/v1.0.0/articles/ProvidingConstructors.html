Providing Constructors

UCalc allows expressions to create objects using the new keyword.

new Vector3(x, y, z)

Constructors are provided through global symbol providers.
If you want to expose constructors for additional types, you need to create and register your own provider.

Creating a Constructor Provider

To provide constructors you need to implement:

  • ISymbolProvider
  • ISymbolEnumerator (for editor autocomplete support)

The provider is then registered globally using:

UCalcContextRegistry.RegisterGlobalProvider(provider);

This makes the constructors available to all expressions.

Using the Project String Comparer

To stay consistent with the rest of UCalc, you should use the string comparer defined in project settings.

UCalcProjectSettings.Instance.TokenComparer

This ensures constructor tokens behave consistently with the configured case sensitivity rules.

Creating Constructor Symbols

To expose a constructor to UCalc you must create an ExpressionConstructor.

The simplest way is to obtain a ConstructorInfo using reflection:

var ctor = typeof(MyType).GetConstructor(new [] { typeof(int), typeof(float) });
var symbol = new ExpressionConstructor(ctor);

Once created, the symbol can be returned by your provider when the constructor token is requested.

Example Provider

Below is a simplified example of a constructor provider that exposes constructors for several Unity types.

internal sealed class UnityConstructorsProvider : ISymbolProvider, ISymbolEnumerator
{
    private readonly Dictionary<string, List<ExpressionConstructor>> m_Constructors;
    private readonly StringComparer m_Comparer;

    public UnityConstructorsProvider(StringComparer comparer)
    {
        m_Comparer = comparer ?? StringComparer.Ordinal;
        m_Constructors = new Dictionary<string, List<ExpressionConstructor>>(m_Comparer);

        PopulateDefaults();
    }

    public bool TryGetArgument(string token, out Expression argOrConst)
    {
        argOrConst = null;
        return false;
    }

    public bool TryGetProperty(string token, out ExpressionProperty parameter)
    {
        parameter = default;
        return false;
    }

    public bool TryGetFunctions(string token, out IReadOnlyList<ExpressionFunction> overloads)
    {
        overloads = null;
        return false;
    }

    public bool TryGetConstructors(string token, out IReadOnlyList<ExpressionConstructor> overloads)
    {
        if (m_Constructors.TryGetValue(token, out var list) && list.Count > 0)
        {
            overloads = list;
            return true;
        }

        overloads = null;
        return false;
    }

    public bool TryGetSubProvider(string token, out ISymbolProvider subProvider)
    {
        subProvider = null;
        return false;
    }
}

This provider stores constructors in a dictionary keyed by the type token.

Populating Constructors

Constructors are typically discovered using reflection:

private void AddConstructors(Type type)
{
    var ctors = type.GetConstructors(BindingFlags.Public | BindingFlags.Instance);

    foreach (var c in ctors)
    {
        if (c.IsStatic) continue;
        if (c.ContainsGenericParameters) continue;

        list.Add(new ExpressionConstructor(c));
    }
}

Each constructor is wrapped in an ExpressionConstructor, which makes it recognizable by the UCalc compiler.

Autocomplete Support

To enable constructor suggestions in the editor, implement the enumeration methods from ISymbolEnumerator.

Example:

public void EnumerateConstructors(List<(string name, IReadOnlyList<ExpressionConstructor> overloads)> dst)
{
    dst.AddRange(m_Constructors.Select(kvp => (kvp.Key, kvp.Value as IReadOnlyList<ExpressionConstructor>)));
}

This allows the editor to display constructor suggestions while typing expressions.

Result

Once registered, constructors become available in expressions:

new Vector3(x, y, z)
new Color(1, 0, 0)

The correct constructor overload is selected automatically based on argument types.
